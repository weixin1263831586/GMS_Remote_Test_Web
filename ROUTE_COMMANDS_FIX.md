# 路由命令说明修正

## 问题说明

原代码中的路由命令说明存在逻辑错误，导致用户混淆应该在哪个主机上执行路由命令。

## 网络拓扑

```
客户端主机 (172.16.14.68)
    ↓ 访问Web界面
测试主机 (172.16.14.233) ← 运行GMS Auto Test，执行路由命令
    ↓ 添加路由 → 172.16.21.0/24 via 172.16.14.1
    ↓
Android设备 (172.16.21.x) ← 设备网段
    ↑
网关/路由器 (172.16.14.1)
```

## 正确的逻辑

### 路由目的

**让测试主机能够访问Android设备网段（172.16.21.x）**

### 执行位置

**在测试主机（172.16.14.233）上执行**

### 路由命令

**Windows:**
```cmd
# 在测试主机上执行以下命令:
# 添加到Android设备网段的路由（通过测试主机网关）
route add 172.16.21.0 mask 255.255.255.0 172.16.14.1
# 检查路由表: route print
# 删除路由: route delete 172.16.21.0
```

**Linux:**
```bash
# 在测试主机上执行以下命令:
# 添加到Android设备网段的路由（通过测试主机网关）
sudo ip route add 172.16.21.0/24 via 172.16.14.1
# 检查路由表: ip route show
# 删除路由: sudo ip route del 172.16.21.0/24
```

def _generate_route_commands(test_network: str, client_network: str, test_host_ip: str, client_ip: str) -> dict:
    """生成路由命令

    网络拓扑说明：

    - 客户端主机: 172.16.14.68 (用户访问Web界面)
    - Android设备: 172.16.21.x (设备网段)
    - 测试主机网关: 172.16.14.1

    路由目的：让测试主机能够访问Android设备网段

    # 推测网关地址（通常是网段的第一个IP）
    test_gateway = '.'.join(test_network.split('.')[:3]) + '.1'

    return {
        'windows': [

            f"# 添加到Android设备网段的路由（通过测试主机网关）",
            f"route add {test_network} mask 255.255.255.0 {test_gateway}",
            f"# 检查路由表: route print",
            f"# 删除路由: route delete {test_network}"
        ],
        'linux': [
            f"# 在测试主机上执行以下命令:",
            f"# 添加到Android设备网段的路由（通过测试主机网关）",
            f"sudo ip route add {test_network}/24 via {test_gateway}",
            f"# 检查路由表: ip route show",
            f"# 删除路由: sudo ip route del {test_network}/24"
        ],
        'note': [
            f"⚠️ 重要提示:",
            f"1. 这些路由命令应该在【测试主机】上执行（172.16.14.233）",
            f"2. {test_gateway} 是测试主机的网关地址（路由器）",
            f"3. 目的：让测试主机能访问Android设备网段 ({test_network})",
            f"4. 如果已经在同一网段，不需要添加路由",
            f"5. 删除路由前请确保不会影响SSH连接"
        ]
    }

## 常见问题

### Q1: 为什么要在测试主机添加路由？

**A**: 因为测试主机需要通过SSH连接到Android设备进行测试操作。

### Q2: 为什么路由指向网关而不是直接连设备？

**A**: 网关（172.16.14.1）是路由器，负责在不同网段之间转发数据包。测试主机需要通过网关才能到达设备网段。

### Q3: 如果测试主机和设备在同一网段呢？

**A**: 如果在同一网段，通常不需要添加路由，因为它们可以直接通信。

### Q4: 如何确认网关地址？

**A**:
- Windows: `ipconfig` 查看"默认网关"
- Linux: `ip route | grep default` 查看默认网关
- 通常是网段的第一个IP（如172.16.14.1）
