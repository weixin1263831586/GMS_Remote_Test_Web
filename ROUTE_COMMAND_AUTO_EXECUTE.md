# 路由命令自动执行功能

## 问题描述

之前的实现中，路由命令只是显示在终端上，但不会自动执行。用户需要手动按回车键才能执行命令，这导致：
1. 用户体验不佳，需要额外操作
2. 可能会忘记执行命令
3. 需要手动复制粘贴命令

**之前的问题**：
```
hcq@ats-041055-64g:~$ sudo ip route add 10.10.10.0/24 via 172.16.14.1
```

命令显示后光标在末尾，但不会自动执行，用户需要手动按回车。

## 解决方案

修改代码，通过WebSocket直接发送完整的命令（包括回车）到服务器执行。

### 关键修改

**文件**: `templates/index_fastapi.html`

```javascript
// 显示路由命令提示信息
terminal.writeln('\x1b[33m========================================\x1b[0m');
terminal.writeln('\x1b[33m📋 路由命令已准备就绪\x1b[0m');
// ... 其他提示信息 ...

// 通过WebSocket直接发送命令到服务器
setTimeout(() => {
    if (terminalSocket && terminalSocket.readyState === WebSocket.OPEN) {
        // 发送命令（包括回车）
        const fullCommand = pendingRouteCommand + '\r';
        terminalSocket.send(JSON.stringify({
            type: 'terminal_input',
            input: fullCommand
        }));
    }
}, 500);  // 延迟500ms确保提示信息已经显示
```

### 工作原理

1. **显示命令**: 使用 `terminal.writeln()` 显示完整的命令行
2. **发送回车**: 通过WebSocket发送 `\r` (回车键) 到服务器
3. **执行命令**: 服务器接收到回车键后执行命令
4. **需要密码**: 如果命令需要sudo权限，会提示用户输入密码

## 技术细节

### WebSocket通信

终端通过WebSocket与服务器通信：

```javascript
// 发送回车键
terminalSocket.send(JSON.stringify({
    type: 'terminal_input',
    input: '\r'  // 回车键的ASCII码
}));
```

### 延迟执行

使用100ms延迟确保：
1. 命令已经完整显示在终端上
2. 用户能看到命令被执行的过程
3. WebSocket连接稳定

```javascript
setTimeout(() => {
    // 发送回车键
}, 100);
```

### 提示文字更新

## 优势

1. **自动化**: 无需手动按回车

2. **网络延迟**: 100ms延迟确保命令正确显示和执行
