# 路由检查功能说明

## 功能概述

"📡 检查路由"按钮已更新为交互式网络连通性测试工具。

## 使用方法

1. **点击"📡 检查路由"按钮**
   - 位置: SSH连接区域
   - 打开路由检查对话框

2. **填写IP地址**
   - **测试主机IP**: 自动从配置文件读取 `ubuntu_host` 的IP部分
   - **客户端IP**: 您当前浏览器的IP地址

3. **点击"🔍 测试连通性"按钮**
   - 系统会测试两个IP之间的网络连通性
   - 检查是否在同一网段
   - 尝试建立SSH连接验证可达性

## 测试结果

### ✅ 连通性测试通过
- 显示: "可连通" 状态
- 延迟信息
- 结论: 网络配置正常，无需添加路由

### ❌ 连通性测试失败
- 显示: "不可连通" 状态
- 可能原因:
  - 客户端和测试主机不在同一网段
  - 缺少必要的路由配置
  - 防火墙阻止了连接

- 建议路由命令:
  - Windows 路由命令
  - Linux 路由命令

## 路由命令示例

### Windows
```batch
route add 192.168.1.0 mask 255.255.255.0 192.168.2.100
route add 192.168.2.0 mask 255.255.255.0 192.168.1.100
route print
route delete 192.168.1.0
route delete 192.168.2.0
```

### Linux
```bash
sudo ip route add 192.168.1.0/24 via 192.168.2.100
sudo ip route add 192.168.2.0/24 via 192.168.1.100
ip route show
sudo ip route del 192.168.1.0/24
sudo ip route del 192.168.2.0/24
```

## 技术实现

### 前端
- **文件**: `static/js/app.js`
- **函数**: `checkRouting()`
- **功能**:
  - 创建模态对话框
  - IP格式验证
  - 调用后端API进行连通性测试
  - 显示测试结果和路由命令

### 后端
- **文件**: `app_fastapi_full.py`
- **API端点**: `POST /api/ssh/route/ping`
- **功能**:
  - 接收测试主机IP和客户端IP
  - 验证IP格式
  - 检查网段
  - 通过SSH连接测试实际可达性
  - 生成路由命令

### 样式
- **文件**: `static/css/style.css`
- **样式类**:
  - `.route-check-dialog` - 对话框容器
  - `.route-check-content` - 对话框内容
  - `.ping-success` - 成功状态样式
  - `.ping-failure` - 失败状态样式
  - `.route-commands` - 路由命令显示区域

## 注意事项

1. **IP地址格式**: 必须是有效的IPv4地址 (例如: 192.168.1.100)
2. **网络权限**: 添加路由命令通常需要管理员权限
3. **防火墙**: 确保防火墙允许相关端口通信
4. **VPN连接**: 如果使用VPN，确保VPN已连接

## 故障排除

### 问题: "IP地址格式不正确"
- 解决: 检查IP地址格式，应为 xxx.xxx.xxx.xxx 格式
- 每个数字范围: 0-255

### 问题: "测试失败"
- 可能原因:
  - 测试主机未启动或网络不可达
  - SSH服务未运行
  - 防火墙阻止连接
- 解决: 检查网络连接和SSH服务状态

### 问题: 测试通过但实际无法连接
- 可能原因:
  - SSH密钥配置问题
  - 端口被占用
  - 权限问题
- 解决: 检查SSH配置和系统日志
