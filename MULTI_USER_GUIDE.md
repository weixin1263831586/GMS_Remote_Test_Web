# GMS Auto Test - 多用户并发访问指南

## 📋 功能说明

本应用已支持多用户同时访问和使用，每个用户的测试状态和日志完全隔离。

## ✨ 多用户特性

### 1. 会话隔离
- 每个用户访问时自动分配唯一的 session_id
- 每个用户拥有独立的测试状态
- 每个用户拥有独立的日志记录

### 2. Socket.IO 房间
- 每个用户加入自己的房间
- 实时日志更新只发送给对应用户
- 支持多用户同时查看不同的测试日志

### 3. 自动清理
- 超过 24 小时的旧会话自动清理
- 防止内存泄漏

## 🚀 使用方式

### 单用户访问
```
http://172.16.14.233:5000/
```

### 多用户同时访问
- 用户A: http://172.16.14.233:5000/
- 用户B: http://172.16.14.233:5000/
- 用户C: http://172.16.14.233:5000/

每个用户可以：
- ✅ 同时选择不同的设备
- ✅ 同时运行不同的测试类型
- ✅ 独立查看自己的测试日志
- ✅ 独立保存自己的日志文件

## 📁 日志文件命名

日志文件按用户和时间戳隔离：
```
logs/
├── cts_20260308_160816_user1.log
├── gts_20260308_161234_user2.log
└── cts_20260308_163045_user3.log
```

## 🔧 技术架构

### 后端
- Flask Session 管理
- 用户状态字典: `user_states[session_id]`
- Socket.IO 房间隔离
- 线程安全的锁机制

### 前端
- 自动携带 session cookie
- Socket.IO 自动连接到个人房间
- 状态完全隔离

## ⚠️ 注意事项

1. **设备冲突**: 多个用户同时选择同一个设备时，后测试会先占用设备
2. **SSH连接**: SSH连接池全局共享，但每个用户的状态独立
3. **日志保存**: 每个用户保存的日志文件独立，不会互相覆盖

## 📊 性能建议

- 建议同时运行测试的用户数不超过 5 个
- 每个用户建议选择不同的设备进行测试
- SSH连接池大小默认为 5

## 🔍 调试

查看当前在线用户：
```bash
# 通过日志查看连接的用户
grep "Client connected" /tmp/flask_final.log
```

查看会话清理日志：
```bash
grep "Cleaned up expired session" /tmp/flask_final.log
```
