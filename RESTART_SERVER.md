# ⚠️ 服务器重启提醒

## 问题说明

您遇到了"测试失败: Request failed"错误，这是因为新添加的API端点需要服务器重启才能生效。

## 解决方案

### 方案1: 重启FastAPI服务器（推荐）

```bash
# 找到当前运行的FastAPI进程
ps aux | grep uvicorn

# 停止当前进程（使用上面找到的PID）
# 例如: kill 50923

# 或者使用pkill
pkill -f "app_fastapi_full"

# 重新启动服务器
cd /home/hcq/GMS_Auto_Test/web_app
python3 -m uvicorn app_fastapi_full:app --host 0.0.0.0 --port 5001 --log-level info --access-log
```

### 方案2: 使用临时后备方案

前端代码已经更新为自动后备方案：
- 如果新的POST API不可用，会自动使用现有的GET API
- 会手动分析IP地址和网段
- 生成相应的路由命令建议

**无需重启服务器即可使用基本功能！**

## 验证服务器状态

重启后，验证新API是否可用：

```bash
curl -X POST http://localhost:5001/api/ssh/route/ping \
  -H "Content-Type: application/json" \
  -d '{"test_host_ip":"172.16.14.233","client_ip":"10.10.10.206"}'
```

如果返回JSON结果（而不是"404 Not Found"），说明服务器已正确重启。

1. **实际连通性测试**: 通过SSH连接验证网络可达性
